## Residual AI-like traits

- Centered LP structure: mostly avoided. The central specimen is physically central, but the page does not use hero copy hierarchy.
- Equal cards: avoided in the first view and mostly avoided below. The lower readout chips still have a grid logic.
- Pill CTA: none.
- Gradient reliance: not dominant. The disc uses color bands, but the page is not built from gradient backgrounds.
- Orb / ellipse / pill escape: mostly avoided. The disc is circular because it is a CD-ROM object, not a decorative orb.
- Uniform same-kind treatment: improved, but the chip system still reveals a generated pattern.
- Top strong / lower ordinary: mostly avoided. Lower sections keep the same material language, though the route table is the most conventional part.
- Reference value: high. The lead screenshot directly shaped the central object-plus-orbit composition.

## Core assessment

Most effective factor:
- RFM workflow forced the lead-reference structure and avoid list.
- The material-first step was the strongest practical lever because the Arecibo SVG and custom object assets prevented a generic panel layout.
- `frontend-design` helped define the subject, palette, and anti-template critique, but it was less decisive than the RFM/material sequence.
- Codex implementation ability mattered in making the single-file responsive page, creating SVG assets, downloading fonts, and fixing screenshot issues.

Material use:
- Successful enough to say the materials are visual leads. The external Arecibo diagram is the central specimen, not a small accent.
- The self-authored disc and pulsar sheet make the page feel like a physical educational object.
- The fonts are important; without Micro 5 the title would lose much of the kiosk/CD-ROM voice.

Could it work without external materials:
- It could function with only self-authored SVGs, but it would be weaker.
- The external Arecibo asset is not absolutely required for layout, but it gives the build a real information-object anchor that CSS alone would not supply.
- The external fonts are close to decisive for the register.

Comparison to the Claude Code "materials suddenly got stronger" phenomenon:
- Partly reproduced. The page changed substantially once the external diagram and local fonts were fixed before layout.
- The strongest material jump came from the Arecibo SVG plus custom SVGs, not from a skill automatically producing visuals.

Future value of RFM in Codex:
- Yes, it is worth using. Codex can follow RFM, gather licensed materials, create local assets, implement the page, and verify screenshots.
- The workflow matters because it prevents Codex from defaulting to a neat LP skeleton.
- The next improvement would be more manual per-element art direction instead of using CSS rules for repeated variation.
